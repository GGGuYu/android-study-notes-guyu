# 音乐与视频播放实现流程对比

## 一句话总结

- **音乐播放**：使用 **Service + MediaSession** 架构，支持后台播放和通知栏控制
- **视频播放**：直接在 **Activity/Fragment** 中使用 **ExoPlayer**，无需后台保活

---

## 一、音乐播放完整流程

### 1. UI 层（MusicPlayerRoute.kt）
```
MusicPlayerRoute.kt
    ↓
MusicPlayerScreen (Composable)
    ↓
用户点击播放/暂停/切歌按钮
```

**作用**：显示音乐播放界面，包含封面、进度条、控制按钮等。

---

### 2. 控制层（ViewModel + MediaServiceConnection）
```
MusicPlayViewModel
    ↓
MediaServiceConnection (单例)
    ↓
MediaController (媒体控制器)
    ↓
发送命令：play() / pause() / seekTo() / skipToNext()
```

**作用**：
- ViewModel 处理业务逻辑
- MediaServiceConnection 创建 MediaController 与 Service 通信
- 所有命令通过 MediaController 远程发送到 MediaService

---

### 3. 服务层（MediaService.kt）
```
MediaService (继承 MediaSessionService)
    ↓
├─ ReplaceableForwardingPlayer (包装层)
    ↓
├─ ExoPlayer (真正的播放器)
    ↓
播放音频文件
```

**作用**：
- **MediaService** 是前台服务（Foreground Service），App 被划掉也能存活
- **MediaSession** 自动创建通知栏，支持后台控制
- **ExoPlayer** 实际解码和播放音频

---

### 4. 数据监听与 UI 更新（回调流程）
```
ExoPlayer (MediaService 中)
    ↓
Player.Listener.onEvents() 回调
    ↓
MediaServiceConnection 更新 StateFlow
    ↓
nowPlaying.value = 当前歌曲
playbackState.value = 播放状态
currentPosition.value = 播放进度
    ↓
MusicPlayerRoute 通过 collectAsStateWithLifecycle() 监听
    ↓
Compose 自动重组，UI 更新
```

**关键类**：
- `MediaServiceConnection.nowPlaying: StateFlow<MediaItem>` - 当前播放歌曲
- `MediaServiceConnection.playbackState: StateFlow<PlaybackState>` - 播放状态
- `MediaServiceConnection.currentPosition: StateFlow<Long>` - 播放进度

---

### 5. 架构示意图

```
┌────────────────────────────────────────────────────────────┐
│                     UI 层 (Compose)                         │
│  MusicPlayerRoute.kt                                       │

│  ├─ 显示进度条：currentPosition.collectAsStateWithLifecycle()│
│  └─ 用户操作：点击播放/暂停/切歌按钮                        │
└───────────────────────┬────────────────────────────────────┘
                        │
                        ▼
┌────────────────────────────────────────────────────────────┐
```

## 二、短视频播放完整流程

### 1. UI 层（ShortVideoRoute.kt）
```
ShortVideoRoute.kt
    ↓
ShortVideoScreen (Composable)
    ↓
VerticalPager (垂直翻页)
    ↓
ItemShortVideo (每个视频项)
```

**作用**：
- 使用 **VerticalPager** 实现抖音式上下滑动
- 每个页面是一个 `ItemShortVideo` Composable
- 通过 `AndroidView` 嵌入原生 `PlayerView`

---

### 2. 控制层（ShortVideoViewModel）
```
ShortVideoViewModel
    ↓
创建 5 个 ExoPlayer 实例（复用池）
    ↓
players[0] ~ players[4] 循环使用
    ↓
onPageChange(index) → 暂停上一个，播放下一个
```

**作用**：
- ViewModel 管理 5 个 ExoPlayer 实例（复用优化）
- 切换页面时：暂停当前播放器，播放新的播放器
- 无需 Service，直接在 ViewModel 中控制

---

### 3. 播放层（ItemShortVideo 中）
```
AndroidView(factory = { context ->
    PlayerView(context).apply {
        this.player = exoPlayer  // 绑定 ExoPlayer
        useController = false    // 不显示默认控制器
        
        // 添加监听器
        player.addListener(object : Player.Listener {
            override fun onIsPlayingChanged(isPlaying: Boolean) {
                // 显示/隐藏暂停按钮
                // 通知父组件播放状态变化
            }
            override fun onPlaybackStateChanged(state: Int) {
                // 更新进度条范围
            }
        })
    }
})
```

**作用**：
- 使用 **AndroidView** 将原生 `PlayerView` 嵌入 Compose
- 通过 `Player.Listener` 监听播放状态
- 直接在 Composable 中更新 UI（暂停按钮、进度条）

---

### 4. 数据监听与 UI 更新（回调流程）
```
ExoPlayer (ItemShortVideo 中)
    ↓
Player.Listener.onIsPlayingChanged() 回调
    ↓
更新 Compose State
    ↓
showPlayButton = !isPlaying
onVideoPlayingChanged(isPlaying)  // 通知父组件
    ↓
Compose 自动重组
    ↓
显示/隐藏暂停按钮、更新进度条
```

---

## 三、关键对比

| 特性 | 音乐播放 | 视频播放 |
|------|----------|----------|
| **播放场景** | 后台播放（锁屏、划掉 App） | 前台播放（必须在界面内） |
| **Service** | ✅ MediaService（前台服务） | ❌ 无需 Service |
| **播放器实例** | 1 个 ExoPlayer（全局唯一） | 5 个 ExoPlayer（复用池） |
| **MediaSession** | ✅ 需要（通知栏控制） | ❌ 不需要 |
| **MediaController** | ✅ 用于跨进程通信 | ❌ 直接使用 ExoPlayer |
| **状态管理** | StateFlow + 全局监听 | Compose State + 局部监听 |
| **架构复杂度** | 高（Service + Connection） | 低（直接使用 ExoPlayer） |

---

## 四、一句话记忆

> **音乐**：Service 保活 + MediaSession 通知栏 + MediaController 远程控制  
> **视频**：ExoPlayer 直接播放 + AndroidView 嵌入 + 局部状态管理
